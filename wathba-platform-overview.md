# منصة وثبة التعليمية
## دليل شامل للمنصة ومميزاتها

---

## نبذة عامة

**وثبة** منصة تعليمية متكاملة (SaaS) مصممة خصيصاً لمدرسي السناتر الخصوصية في مصر. تمنح كل مدرس منصته التعليمية الخاصة بهويته البصرية ورابطه المميز، وتتيح له إدارة كورساته وطلابه وامتحاناته ومدفوعاته من مكان واحد بشكل احترافي.

## 1. مفهوم المنصة

### نموذج Multi-Tenant
كل مدرس يحصل على بيئة عمل مستقلة تماماً:

- **رابط مخصص** على شكل `/اسم-المدرس` (مثال: `/admin`، `/ms-sara`)
- **اسم المنصة** الخاص به (مثال: "أكاديمية محمد للرياضيات")
- **شعار وصورة شخصية** خاصة
- **عزل تام للبيانات** — طلاب كل مدرس ومحتواه لا يظهر للآخرين

### من يستخدم المنصة؟
| الجهة | الوصف |
|-------|-------|
| مدرسو السناتر | المستخدم الأساسي — يملك كامل الصلاحيات |
| المساعدون | فريق مساعد بصلاحيات محددة من المدرس |
| الطلاب | يتفاعلون مع الكورسات والامتحانات |

## 2. الأدوار والصلاحيات

### المدرس (Teacher)
يملك صلاحيات كاملة على كل عناصر منصته:
- إنشاء وتعديل الكورسات والامتحانات
- إدارة الطلاب والمساعدين
- رؤية كل التحليلات والتقارير
- إرسال الإشعارات
- التحكم في البث المباشر
- تصدير البيانات واستيرادها

### المساعد (Assistant)
صلاحيات قابلة للتخصيص بشكل دقيق — المدرس يفعّل أو يعطّل كل صلاحية على حدة:

| الصلاحية | الوصف |
|----------|-------|
| إضافة طلاب | إمكانية تسجيل طلاب جدد |
| تعديل الطلاب | تحديث بيانات الطلاب |
| حذف الطلاب | إزالة الطلاب من المنصة |
| إدارة الامتحانات | إنشاء وتعديل الامتحانات |
| عرض التحليلات | الاطلاع على إحصائيات الأداء |
| إرسال التقارير | تصدير وإرسال تقارير الطلاب |
| إدارة المدفوعات | التحقق من عمليات الدفع |
| إدارة الكورسات | إضافة وتعديل محتوى الكورسات |
| إرسال الإشعارات | التواصل مع الطلاب وأولياء الأمور |

### الطالب (Student)
- يشاهد الكورسات المسجّل فيها فقط
- يؤدي الامتحانات ويراجع نتائجه
- يرى ترتيبه في المتصدرين
- يشارك في الفعاليات والألعاب

## 3. إدارة الكورسات والمحتوى

### هيكل الكورس
كل كورس يتكون من طبقات منظمة:

```
الكورس
 ├── القسم الأول (مثال: الباب الأول)
 │    ├── فيديو 1
 │    ├── فيديو 2
 │    └── ملف PDF
 ├── القسم الثاني
 │    ├── فيديو 3
 │    └── ملف PDF
 └── امتحان مرتبط بالكورس
```

### خصائص الكورس
- **الاسم والوصف** التفصيلي
- **صورة مصغرة (Thumbnail)** جذابة
- **السعر** — مدفوع أو مجاني تماماً
- **المرحلة الدراسية المستهدفة** (أول / ثاني / ثالث ثانوي، إلخ)
- **نشر / إخفاء** الكورس عن الطلاب
- **نقاط مكافأة** عند إتمام الكورس

### رفع المحتوى
| النوع | التفاصيل |
|-------|----------|
| فيديوهات | رفع مباشر أو رابط خارجي (YouTube) |
| جودة متعددة | دعم 480p / 720p / 1080p |
| ملفات PDF | رفع مباشر مع حماية بـ JWT |
| حماية المحتوى | الفيديوهات والـ PDF محمية — تتطلب تسجيل دخول |

### متابعة تقدم الفيديو
- تتبع تلقائي للدقائق المشاهَدة لكل فيديو
- حساب النسبة المئوية للتقدم
- حفظ آخر موضع للتشغيل (Resume)
- عداد عدد مرات المشاهدة

## 4. منظومة الامتحانات

### أنواع الأسئلة
| النوع | الوصف |
|-------|-------|
| اختيار من متعدد (MCQ) | 4 خيارات، إجابة واحدة صحيحة |
| صح / خطأ (True/False) | سؤال بإجابتين |
| مقالي (Essay) | إجابة حرة يصحّحها المدرس يدوياً |

### إعدادات الامتحان المتقدمة
- **مدة الامتحان** بالدقائق مع مؤقت تلقائي
- **تاريخ البداية والنهاية** — الامتحان يفتح ويغلق تلقائياً
- **درجة النجاح** القابلة للضبط
- **خلط الأسئلة** (Shuffle) عشوائياً
- **خلط الخيارات** لمنع الغش
- **نقاط عند المحاولة** — حتى لو راسب
- **نقاط إضافية عند النجاح**
- **الشارة (Badge)** — اسم ولون خاص للمتفوقين

### بنوك الأسئلة
- إنشاء بنك أسئلة مرتبط بكورس
- سحب عدد محدد من الأسئلة عشوائياً لكل طالب
- استخدام نفس البنك في امتحانات متعددة

### حماية الامتحانات (Anti-Cheat)
- **جلسة امتحان Server-Side** — المؤقت يُحسب على السيرفر
- لا يمكن التلاعب بالوقت من المتصفح
- الأسئلة تُخزَّن في snapshot على السيرفر

### تجربة الطالب في الامتحان
- مؤقت واضح في الواجهة
- التنقل بين الأسئلة بحرية
- تقديم الامتحان قبل انتهاء الوقت
- **مراجعة فورية** بعد التسليم — يرى الصح والخطأ

### طلب إعادة الامتحان
- الطالب يرسل طلب مع سبب
- المدرس يوافق أو يرفض مع إضافة ملاحظة
- إشعار تلقائي للطالب بالقرار

## 5. إدارة الطلاب

### بيانات الطالب
- الاسم الكامل والرقم التعريفي
- رقم هاتف الطالب ورقم ولي الأمر
- المرحلة الدراسية والنوع
- تاريخ التسجيل والنشاط الأخير
- إجمالي النقاط المتراكمة

### عمليات إدارة الطلاب
- إضافة طالب يدوياً
- **استيراد جماعي** (Bulk Import) من ملف Excel/CSV
- بحث وفلترة متقدمة
- **حذف ناعم (Soft Delete)** — يمكن استعادة الطالب
- تصدير قائمة الطلاب بصيغ متعددة

### طلبات التسجيل في الكورسات
- الطالب يرسل طلب انضمام مع صورة إيصال الدفع
- المدرس / المساعد يراجع ويوافق أو يرفض
- عند الموافقة: التسجيل يتم تلقائياً

### بروفايل الطالب التفصيلي
المدرس يرى لكل طالب:
- الكورسات المسجّل فيها وتقدمه في كل كورس
- تاريخ الامتحانات ودرجاته
- الشارات المكتسبة
- منحنى الأداء عبر الزمن
- مقارنة أدائه بمتوسط الفصل

## 6. نظام المدفوعات

### وسائل الدفع المدعومة
- InstaPay
- Vodafone Cash
- Fawry
- أي وسيلة أخرى (النظام مرن)

### سير عملية الدفع
```
الطالب يدفع → يرفع صورة الإيصال →
طلب "Pending" → المدرس/المساعد يتحقق →
Verified أو Rejected → التسجيل يتم تلقائياً
```

### حالات الدفع
| الحالة | المعنى |
|--------|--------|
| Pending | بانتظار التحقق |
| Verified | مؤكّد ومسجّل |
| Rejected | مرفوض |

### تقارير المدفوعات
- إجمالي الإيرادات لكل كورس
- تاريخ عمليات الدفع
- المدفوعات المعلّقة التي تحتاج مراجعة
- كشف حساب مفصّل

## 7. البث المباشر

### تقنية الإرسال
المنصة تستخدم **LiveKit** لجلسات البث المباشر — بث فيديو وصوت عالي الجودة مع تحكم كامل في الصلاحيات.

### خصائص البث
- إنشاء غرفة بث بعنوان ووصف
- **التحكم في الوصول:**
  - مفتوح لكل الطلاب
  - مقيّد بمرحلة دراسية معينة
  - مقيّد بطلاب محددين بالاسم
- **قفل الجلسة** — منع الانضمام الجديد بعد البدء

### أدوات أثناء البث
| الأداة | الوصف |
|--------|-------|
| الشات المباشر | رسائل نصية أثناء الجلسة |
| رفع اليد | الطالب يطلب الكلام |
| إدارة المتحدثين | المدرس يمنح/يسحب إذن الكلام |
| مشاركة الشاشة | المدرس أو الطالب (بإذن) |
| طرد المشاغبين | إزالة طالب من الجلسة |

### جدولة البث
- تحديد موعد مسبق للجلسة
- الطلاب يرون المواعيد القادمة

## 8. الإشعارات والتواصل

### قنوات الإشعارات
| القناة | النوع |
|--------|-------|
| إشعار داخل المنصة | فوري عبر SSE (Server-Sent Events) |
| Push Notifications | عبر Firebase Cloud Messaging |
| WhatsApp | رسائل مباشرة لرقم الطالب/ولي الأمر |

### أنواع الإشعارات التلقائية
- نشر امتحان جديد
- نتيجة امتحان
- الموافقة / الرفع على طلب الانضمام
- قبول / رفض طلب إعادة الامتحان
- بدء جلسة بث مباشر

### إشعارات يدوية
- المدرس يختار المستلمين (الكل / مرحلة / طالب بعينه)
- اختيار نوع الإشعار والرسالة
- يمكن إرسالها لولي الأمر أيضاً

## 9. التحليلات والتقارير

### تحليلات المدرس (Dashboard)
- إجمالي الطلاب النشطين
- عدد الكورسات والامتحانات
- إيرادات المدفوعات المؤكّدة
- الطلاب الجدد هذا الشهر

### تحليلات الأداء
- متوسط درجات الامتحانات لكل كورس
- نسبة النجاح والرسوب
- أكثر الأسئلة خطأً (تحديد نقاط الضعف)
- تقدم الطلاب في مشاهدة الفيديوهات

### تقارير قابلة للتصدير
- قائمة الطلاب كاملة (PDF / Excel)
- نتائج الامتحانات مع التفاصيل
- كشف حسابات المدفوعات
- تقرير أداء طالب واحد

### بروفايل الطالب التحليلي
- رسم بياني لمسيرة الدرجات
- مقارنة الأداء بين الامتحانات
- نسبة إكمال كل كورس
- آخر نشاط على المنصة

## 10. نظام النقاط والمتصدرين

### كيف يكسب الطالب النقاط؟

| الحدث | النقاط |
|-------|--------|
| محاولة أي امتحان | نقاط المحاولة (قابلة للضبط) |
| النجاح في الامتحان | نقاط إضافية (قابلة للضبط) |
| إتمام كورس كامل | نقاط الإتمام (قابلة للضبط) |
| الفعاليات الأسبوعية | نقاط بحسب الأداء |

### لوحة المتصدرين
- ترتيب تنازلي لكل طلاب المدرس
- تمييز المراكز الثلاثة الأولى بصرياً
- الطالب يرى موضعه دائماً حتى لو في المركز الأخير

### الشارات (Badges)
- كل امتحان يمكن ربطه بشارة (اسم + لون)
- الطلاب المتفوقون (الدرجة الكاملة أو فوق حد معين) يحصلون عليها
- الشارات تظهر في بروفايل الطالب

### إعادة ضبط المتصدرين
- المدرس يضبط دورة الإعادة (شهرية مثلاً)
- عند الإعادة: يُحفظ الترتيب في السجل التاريخي
- الطلاب يمكنهم مشاهدة نتائج الأشهر السابقة

## 11. الألعاب التعليمية

### لعبة Stickman Run
لعبة تعليمية تفاعلية مبنية بـ Canvas:

- **المفهوم:** شخصية تجري وتقفز وتتحاشى العقبات
- **مراحل:** تتصاعب صعوبة الأسئلة مع التقدم
- **Boss Fights:** مواجهة "المدرس الغاضب" بالإجابة على أسئلة
- **الأسئلة:** مصممة حسب المرحلة الدراسية للطالب
- **أسبوعية:** كل طالب يلعب مرة واحدة في الأسبوع
- **الجائزة:** نقاط تُضاف للمتصدرين

### فعاليات أسبوعية
- المدرس يمكنه إضافة فعاليات جديدة
- النتائج تُسجَّل ويمكن مقارنتها
- تحفّز الطلاب على الانخراط المستمر

## 12. لوحة الطالب

### الصفحة الرئيسية للطالب
يرى الطالب فور دخوله:
- إجمالي نقاطه وترتيبه بين الزملاء
- الكورسات المسجّل فيها ونسبة إكمال كل منها
- الامتحانات القادمة مع العد التنازلي
- آخر نشاط له على المنصة

### كورسات الطالب
- قائمة الكورسات المتاحة والمسجّلة
- مشاهدة الفيديوهات مع حفظ التقدم
- تحميل ملفات PDF
- أداء الامتحانات المرتبطة بالكورس

### إحصائيات الطالب الشخصية
- منحنى الدرجات عبر الزمن
- الامتحانات المؤدّاة والنتائج
- الشارات المكتسبة
- المقارنة مع متوسط المجموعة

## 13. المواصفات التقنية

### التقنيات المستخدمة
| الطبقة | التقنية |
|--------|---------|
| Frontend | React 18 + Vite 5 + Tailwind CSS 3 |
| Backend | Node.js 20 + Express 4 |
| قاعدة البيانات | PostgreSQL (Replit Managed) |
| المصادقة | JWT + bcryptjs |
| التحديثات اللحظية | Server-Sent Events (SSE) |
| الإشعارات | Firebase Cloud Messaging (FCM) |
| الرسوم البيانية | ECharts |
| PDF | jsPDF + jspdf-autotable |
| رفع الملفات | Multer → /uploads/ |
| البث المباشر | LiveKit |

### الأمان
- كل الـ API محمية بـ JWT
- ملفات PDF والفيديوهات تتطلب توكن صالح
- حماية من الطلبات المتكررة (Rate Limiting: 120 طلب/دقيقة)
- HTTP Security Headers عبر Helmet.js
- CORS مضبوط بدقة

### الأداء
- فهارس قاعدة البيانات على كل الاستعلامات المتكررة
- Cache لصلاحيات المساعدين
- تحميل تدريجي للمحتوى
